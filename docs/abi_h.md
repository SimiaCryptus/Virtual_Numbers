# `include/nam/abi.h` — The Frozen Automaton ABI

## Overview

This header defines the **frozen, C-compatible Application Binary
Interface (ABI)** for the *automaton tier* of the NAM system. It is the
contract at the boundary between low-level, machine-representable
state and any higher-level (C++ or otherwise) logic built on top of it.

The design deliberately restricts itself to **Plain Old Data (POD)**
in the C sense: every type declared here is *trivially copyable* and
has *standard layout*. This is not an incidental property — it is the
central guarantee the file exists to provide.

## Background and Theory

### Numbers as Automata

The central idea reflected in this ABI is the representation of numbers
— in particular, **algebraic numbers** and **eventually-periodic
expansions** — as the states of a finite automaton or transducer that
emits digits. This view has deep roots in several established fields:

- **Automatic sequences and finite automata.** The theory of
  *automatic sequences* (Allouche & Shallit, *Automatic Sequences:
  Theory, Applications, Generalizations*, 2003) studies sequences
  generated by finite-state machines reading the base-`k`
  representation of an index. The `base` field of `AutomatonVM`
  makes the choice of numeration system (the codec) an explicit,
  external selector rather than something "baked into the number."

- **Continued fractions and periodic orbits.** By Lagrange's theorem,
  a real number has an *eventually periodic* continued fraction
  expansion *if and only if* it is a quadratic irrational. More
  generally, degree-`d` algebraic numbers arise naturally from
  periodic orbits of certain dynamical systems and substitution
  maps. The `phase` field encodes the position within such a
  periodic orbit, and `state[4]` is over-provisioned to hold the
  coordinates required for up to **degree-4 algebraic** state.

- **Transducers and digit streams.** The `NumVMFn` typedef models a
  *single transition* of a sequential transducer: given the current
  machine state, it emits one `digit` and returns the `next` state.
  This is the classic Mealy-machine formulation (Mealy, 1955), where
  output is a function of both state and (implicit) transition.

### Why POD and Trivial Copyability Matter

A recurring operation in this domain is **forking** a computation:
duplicating the current state so that two continuations may proceed
independently (e.g. exploring alternative digit expansions, or
memoizing a periodic orbit). The ABI is engineered so that:

> **fork is a literal struct copy, O(1), with no hidden state.**

This mirrors the persistent / immutable data-structure philosophy
(cf. Okasaki, *Purely Functional Data Structures*, 1998), where cheap
structural sharing and copying enable clean backtracking and
speculative evaluation. Because `AutomatonVM` contains no pointers,
no ownership, and no indirection, copying it is a `memcpy` — there is
nothing to deep-copy and nothing that can alias.

## ABI Stability Rules

The file enforces a strict set of rules (Phase 1, Section 2):

1. **Plain C only.** No C++ constructs may appear above the ABI line.
   The `extern "C"` guard ensures C++ translation units consume these
   declarations with C linkage and C layout.
2. **Versioned layout.** Any change to the layout of these structs
   **must** bump `NAM_ABI_VERSION`. Fields must **never** be silently
   reordered.
3. **Fixed size.** `sizeof(AutomatonVM) == 40`, computed as
   `4 (base) + 4 (phase) + 4 * 8 (state[4])`. This invariant should
   be asserted at compile time by any consumer.

## Versioning

```c
#define NAM_ABI_VERSION 1
```

`NAM_ABI_VERSION` is the single source of truth for the ABI revision.
It must be incremented on **any** change to the byte layout of the
types below, allowing downstream code and serialized artifacts to
detect and reject incompatible representations.

## Type Reference

### `AutomatonVM`

The complete, self-contained state of an automaton-tier virtual
machine.

```c
typedef struct
{
    uint32_t base;     /* codec selector -- NOT baked into the number */
    uint32_t phase;    /* periodic-orbit phase / index                */
    uint64_t state[4]; /* over-provisioned for degree-4 algebraic     */
} AutomatonVM;
```

| Field   | Type          | Offset | Meaning                                                                                                             |
|---------|---------------|--------|---------------------------------------------------------------------------------------------------------------------|
| `base`  | `uint32_t`    | 0      | Numeration-system / codec selector. Kept *outside* the number so the same state can be rendered in different bases. |
| `phase` | `uint32_t`    | 4      | Index/phase within a periodic orbit (cf. Lagrange periodicity).                                                     |
| `state` | `uint64_t[4]` | 8      | Algebraic state coordinates, sized for degree ≤ 4.                                                                  |

**Layout guarantees:** standard-layout, trivially copyable,
`sizeof == 40`.

### `NumVMStep`

The result of advancing the machine by one step: the emitted digit
together with the successor state.

```c
typedef struct
{
    uint32_t   digit;
    AutomatonVM next;
} NumVMStep;
```

| Field   | Type          | Meaning                                          |
|---------|---------------|--------------------------------------------------|
| `digit` | `uint32_t`    | The output symbol emitted by this transition.    |
| `next`  | `AutomatonVM` | The successor state (itself trivially copyable). |

### `NumVMFn`

A pointer to a transition function — the Mealy-style step relation of
the transducer.

```c
typedef NumVMStep (*NumVMFn)(AutomatonVM);
```

Given a state, it produces a `(digit, next-state)` pair. Because the
state is passed and returned **by value**, transition functions are
pure with respect to the ABI: no aliasing, no shared mutable state.

## Usage Notes

- Consumers should statically assert the size invariant, e.g.
  `_Static_assert(sizeof(AutomatonVM) == 40, "ABI drift");`.
- Treat copies of `AutomatonVM` as fully independent values; forking a
  computation is exactly `AutomatonVM forked = original;`.
- Never persist or transmit these structs without recording
  `NAM_ABI_VERSION` alongside them.
