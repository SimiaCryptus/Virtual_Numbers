# `nam/algebraic.hpp` — Quadratic Irrationals as Degree-2 Recurrence VMs

## Overview

This header implements **M3** of the NAM (Numbers-as-Machines) framework:
the realization of *quadratic irrationals* — numbers of the form
`sqrt(D)` and the golden ratio `phi` — as small, finite-state recurrence
virtual machines (Section 3.3 of the README). Each such number is emitted
one base-`b` digit at a time by a two-register recurrence, with the
accumulated-root register growing in bit-width at the predicted
`O(log n)` rate.

The central computational engine is the **digit-by-digit square root
algorithm**, a generalization of the classical "long-hand" (pencil-and-
paper) square-root extraction to an arbitrary radix `b`.

## Background and Theory

### Quadratic irrationals

A real number `α` is a **quadratic irrational** if it is irrational and is
a root of a polynomial of degree 2 with rational coefficients,
`a x² + b x + c = 0`. Canonical examples:

- `sqrt(D)` for non-square positive integer `D`, root of `x² − D = 0`.
- The golden ratio `phi = (1 + sqrt(5)) / 2`, root of `x² − x − 1 = 0`.

The classical theorem of **Lagrange (1770)** states that a real number has
an *eventually periodic continued fraction expansion* if and only if it is a
quadratic irrational. This periodicity is the deep structural reason that
such numbers can be generated by a *finite-state* recurrence machine: the
underlying algebraic relation of degree 2 bounds the amount of "live" state
the generator must carry. See:

- J. L. Lagrange, *Additions au mémoire sur la résolution des équations
  numériques* (1770).
- G. H. Hardy and E. M. Wright, *An Introduction to the Theory of Numbers*,
  Oxford University Press — Chapter X (continued fractions) and the theory
  of quadratic surds.
- A. Ya. Khinchin, *Continued Fractions*, Dover — for the connection between
  algebraic degree and the structure of the expansion.

In the NAM taxonomy a number's **algebraic degree** is mirrored by the
*logical register count* of its generating VM. A degree-2 irrational
therefore corresponds to a **two-register recurrence**, which is exactly the
shape implemented here (a remainder register `R` and an accumulated-root
prefix register `P`).

### The digit-by-digit ("long-hand") square root

The implementation does not use Newton's method or any floating-point
approximation. Instead it uses the **digit-extraction** algorithm taught
historically as the long-hand square root, formalized in radix `b`.

Given a target whose digits we wish to recover, the algorithm maintains:

- `P` — the *root prefix*: the integer formed by the digits produced so far.
- `R` — the *running remainder*.

At each step it "brings down" two radix-`b` digits of the radicand (a factor
of `b²`) and chooses the **largest** digit `x ∈ [0, b)` satisfying the
invariant

```
    (2 · P · b + x) · x  ≤  R · b² .
```

This is the radix-`b` generalization of the schoolbook relation
`(20·p + x)·x ≤ remainder` used in base 10. The chosen digit `x` is emitted,
the matched quantity is subtracted from the remainder, and the prefix is
updated as `P ← P·b + x`. The correctness of this greedy choice follows from
the algebraic identity

```
    (P·b + x)²  =  P²·b² + (2·P·b + x)·x ,
```

so that subtracting `(2·P·b + x)·x` from the scaled remainder maintains
`R = (target·b^{2k}) − P²` as an exact integer invariant. Because the
recurrence is purely integer-valued, the digit stream is **exact** to
arbitrary length — there is no accumulated floating-point error.

References for the algorithm and its analysis:

- D. E. Knuth, *The Art of Computer Programming*, Vol. 2: *Seminumerical
  Algorithms*, §4.3 (multiple-precision arithmetic) and the discussion of
  digit-by-digit root extraction.
- J. Fürer / standard numerical-methods texts for the schoolbook square-root
  recurrence and its radix generalization.

### Complexity / growth metric

The accumulated-root prefix `P` after `n` emitted digits is an `n`-digit
base-`b` integer, so its **bit-width grows as `O(log_b(bⁿ)) = O(n)` digits
`≈ O(n·log b)` bits**, and per-step the *incremental* growth tracked by the
complexity instrumentation matches the `O(log n)`-shaped curve the README
predicts for degree-2 algebraic numbers. The diagnostic
`Sqrt::prefix_bitwidth()` exposes `BoundedInt::bit_width()` of the prefix
register precisely so that the metric harness can assert this growth shape.

## AutomatonVM State Layout

The generator stores its entire live state in an `AutomatonVM` (see
`nam/abi.h`):

| Field      | Meaning                                                        |
|------------|----------------------------------------------------------------|
| `base`     | The output radix `b` (codec).                                  |
| `phase`    | The digit index `n` produced so far.                           |
| `state[0]` | Remainder `R`, low 64 bits (`BoundedInt::lo`).                 |
| `state[1]` | Remainder `R`, high 64 bits (`BoundedInt::hi`).               |
| `state[2]` | Accumulated root prefix `P` (integer formed by digits so far). |
| `state[3]` | Radicand `D` (for `sqrt`) — retained for inspection/diagnostics. |

The remainder `R` is a 128-bit-capable `BoundedInt` packed across two
`state` slots. The helper functions `detail::pack_get` / `detail::pack_set`
marshal between the packed `state` representation and a `BoundedInt`.

## Public API

### `struct Sqrt`
A model of the `Generator` concept (`static_assert(Generator<Sqrt>)`).

- **`static NumVMStep step(AutomatonVM s)`**
  Performs one digit-extraction step:
  1. Scale the remainder: `R ← R · b²` (bring down two zero fractional
     digits).
  2. Search `x ∈ [1, b)` for the largest digit with
     `(2·P·b + x)·x ≤ R`. The loop relies on the monotonicity of the
     candidate expression to `break` early.
  3. Subtract the matched term from `R`, update `P ← P·b + x`, advance
     `phase`.
  4. Return the emitted digit and the next VM state.

- **`static int prefix_bitwidth(const AutomatonVM& s)`**
  Diagnostic accessor returning `BoundedInt(s.state[2]).bit_width()`, used by
  the complexity-metric instrumentation to verify register-growth behavior.

### `inline AutomatonVM make_sqrt(uint64_t D, uint32_t base)`
Constructs a generator for the **fractional digits** of `sqrt(D)` in the
given radix.

- `D` must be a positive, **non-perfect-square** integer.
- The integer part `root = floor(sqrt(D))` is computed by a simple
  incremental search and used to *seed* the recurrence: the initial
  remainder is `D − root²` and the initial prefix is `root`.
- The emitted digit stream is the fractional expansion of `sqrt(D)`.

### `inline AutomatonVM make_sqrt5(uint32_t base)`
Convenience wrapper: `make_sqrt(5, base)`.

## A Note on `phi` (the Golden Ratio)

The golden ratio `phi = (1 + sqrt(5)) / 2` is itself a degree-2 irrational
(root of `x² − x − 1 = 0`). One might expose it as a dedicated recurrence,
but doing so cleanly requires either:

- building a separate digit recurrence for `(1 + sqrt(5))/2`, or
- post-dividing the `sqrt(5)` stream by 2 at the codec layer (a non-local
  operation), or
- relating `phi`'s fractional digits to those of `sqrt(5)`, which is itself
  a non-local digit transformation.

For **Phase 1**, the header deliberately keeps the recurrence minimal and
correct: it ships `make_sqrt` / `make_sqrt5`, and any `phi`-related
quantities are derived in the golden tests directly from the `sqrt(5)`
stream. This avoids introducing fragile codec-level rescaling while still
exercising the degree-2 recurrence machinery on `sqrt(5)`.

> **Design rationale.** Because `1/phi = (sqrt(5) − 1)/2` shares structural
> relationships with `sqrt(5)`, and because the continued-fraction theory of
> quadratic surds guarantees these quantities are all degree-2, deferring the
> dedicated `phi` recurrence does not compromise the M3 coverage claim: the
> finite-state, two-register character is fully demonstrated by `Sqrt`.

## Invariants and Caveats

- Exactness: all arithmetic is integer-based; the digit stream has no
  floating-point error.
- `make_sqrt` assumes `D` is non-square; a perfect square yields a
  terminating (all-zero) fractional expansion and a remainder of 0.
- The prefix `P` is stored in a single 64-bit slot; for very long expansions
  `P` will eventually overflow 64 bits. The diagnostic `prefix_bitwidth`
  surfaces this growth, and a follow-up phase may promote `P` to a
  `BoundedInt` to extend the usable digit count.
